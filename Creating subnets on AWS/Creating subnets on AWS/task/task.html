[TITLE]Creating subnets on AWS[/TITLE]

Create a subnet within the Hyper-VPC in your AWS account with the following properties:

  • Name: Hyper-Subnet

  • IPv4 CIDR block:172.16.0.0/28;

  • Availability Zone: us-east-1a

  • Auto-assign public IPv4 address: Enabled

Once you've created the subnet, use the following AWS CLI command to retrieve information about it. This will provide a JSON output with the relevant information. Provide the JSON output as the answer to this task. The --query parameter is structured to retrieve the subnet name, IPv4 CIDR block, availability zone, and whether auto-assign public IPv4 address is enabled:

# if you use PowerShell as your shell, replace backslashes (\) with backticks (`) as line seperators

aws ec2 describe-subnets \
  --filters "Name=vpc-id,Values=<vpc-id>" "Name=tag:Name,Values=Hyper-Subnet" \
  --query 'Subnets[].{Name:Tags[?Key==`Name`].Value | [0], IPv4CIDR:CidrBlock, AvailabilityZone:AvailabilityZone, MapPublicIpOnLaunch:MapPublicIpOnLaunch}' \
  --region us-east-1

Replace <vpc-id> with the actual VPC ID of the "Hyper-VPC." You can get it from the console or using the following command:

aws ec2 describe-vpcs --filters "Name=tag:Name,Values=Hyper-VPC" --query 'Vpcs[0].VpcId' --output text --region us-east-1
[ALERT-warning]

If you get empty braces as output, then the subnet does not exist within the specified VPC

[/ALERT]
